💡 Points intéressants à noter
🏷️
Double tag SHA + branche
Chaque image reçoit deux tags : le SHA court du commit (:a3f7c2d) pour la traçabilité exacte, et le nom de branche (:main) pour le déploiement Helm.
🛡️
Trivy scan AVANT le publish
Les .tar.gz sont scannés au stage suivant avant tout push au registry. Une CVE CRITICAL rend le badge orange mais ne bloque pas.
⚡
Standalone pour Minikube local
En environnement développement, c'est l'image standalone qui est déployée en 3 réplicas dans Kubernetes via Helm, évitant la complexité multi-conteneurs.
📉
Expiration 1 jour
Les artifacts .tar.gz expirent après 24h dans GitLab. Ils servent uniquement de relais vers le stage publish — les images permanentes vivent dans le registry.
