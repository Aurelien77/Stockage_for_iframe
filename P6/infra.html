ansible-playbook deploy-microcrm.yml
Point d'entrée unique — exécuté depuis le poste développeur ou la CI

-e "env=staging"
Avec variable env=staging
🟢
DÉVELOPPEMENT
Image locale Minikube · values.yaml · NodePorts 30080–82
env=dev (défaut)
1
🐳
Démarrage Docker Desktop
Vérifie que Docker est actif. Bloque jusqu'au démarrage complet du daemon.
2
☸️
Démarrage Minikube
Lance le cluster local si absent. Contextualise kubectl.
minikube start --driver=docker
3
🔨
Build image dans Minikube
Construit l'image directement dans le daemon Minikube — pas de registry nécessaire.
eval $(minikube docker-env)
docker build -t microcrm-standalone:latest .
🟢 Dev uniquement
4
🏗️
Terraform — provisionnement
Crée le namespace, le secret PostgreSQL, le PVC et le pod postgres dans Kubernetes.
terraform init && terraform apply -auto-approve
# → namespace microcrm, secret pg, PVC 1Gi
5
⛵
Helm — déploiement avec values.yaml
Déploie les 3 réplicas standalone avec l'image locale, sans imagePullSecrets.
helm upgrade --install microcrm ./helm/microcrm \
  -f values.yaml ← image locale, NodePorts 30080-82
  -n microcrm
🟢 values.yaml — image: microcrm-standalone:latest
6
🐍
Script Python — attente & port-forward
Lance start-microcrm.py : attend les pods Ready, ouvre les port-forwards et vérifie les URLs HTTP.
python3 scripts/start-microcrm.py
✅ Résultat
🌐 App disponible:8888
☸️ 3 réplicas microcrmNodePorts 30080–82
🗄️ PostgreSQLPVC monté
📊 Kibana ELK:5601
🔵
STAGING
Registry GitLab · values-staging.yaml · NodePorts 30083–85
-e "env=staging"
1
🐳
Démarrage Docker Desktop
Même vérification daemon Docker. Identique au mode dev.
2
☸️
Démarrage Minikube
Lance le cluster. Même comportement que dev.
minikube start --driver=docker
—
🚫
Build image — ignoré
Pas de build local. L'image est tirée depuis le GitLab Registry au moment du déploiement Kubernetes.
🔵 Non exécuté en staging
3
🔑
Création imagePullSecret
Crée le secret Kubernetes pour s'authentifier au registry GitLab privé avant le déploiement Helm.
kubectl create secret docker-registry gitlab-registry \
  --docker-server=registry.gitlab.com \
  --docker-username=$GL_USER --docker-password=$GL_TOKEN
🔵 Staging uniquement
4
🏗️
Terraform — provisionnement
Identique au mode dev — namespace, secret PostgreSQL, PVC, pod postgres.
terraform apply -auto-approve
5
⛵
Helm — déploiement avec values-staging.yaml
Kubernetes tire l'image depuis le registry GitLab via imagePullSecrets.
helm upgrade --install microcrm ./helm/microcrm \
  -f values-staging.yaml ← registry GitLab, 30083-85
  -n microcrm
🔵 values-staging.yaml — image: registry.gitlab.com/…/standalone:main
6
🐍
Script Python — attente & port-forward
Identique — attend que les pods soient Ready avant de déclarer le déploiement terminé.
python3 scripts/start-microcrm.py
✅ Résultat
🌐 App disponible:8888
☸️ 3 réplicas microcrmNodePorts 30083–85
🐳 Image depuis registry:main
🔑 imagePullSecretconfiguré
