Build
Stage 2 — conditionnel au succès des tests
🎨 Build Frontend — Node.js / Angular

Angular B · Node.js · Webpack · Terser (minification)

$ cd back Dossier Spring Boot
$ ./gradlew Build Compilation + package du JAR Spring Boot
Gradle Build · Java 17 · Gradle Wrapper · Spring Boot 3.2.5
⚠️ Limitations du pipeline actuel
🔴 Ce qui manque dans ce pipeline à 2 stages
🔍
Analyse qualité du code absente
Aucun SonarQube, ESLint, Checkstyle ni Quality Gate. La dette technique s'accumule sans détection automatique.
🔐
Aucun scan de sécurité dans le pipeline
Trivy est appliqué manuellement après pull par l'équipe Ops — trop tard pour bloquer une image vulnérable avant publication.
🐳
Pas de packaging Docker automatisé
Les images Docker sont construites et publiées manuellement sur Docker Hub. Aucune image versionnée automatiquement.
🚀
Aucune livraison continue (CD)
Pas de déploiement automatique vers staging ou production. Tout déploiement est déclenché manuellement par l'équipe Ops.
🏷️
Versionnage manuel
Pas de Semantic Release ni de Conventional Commits. Les numéros de version sont gérés manuellement (v1.2.3 en dur).
🔑
Détection de secrets absente
Aucun scan Gitleaks. Des credentials pourraient être exposés accidentellement dans le code source sans détection.
