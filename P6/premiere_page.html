Communication manuelle entre DEV et OPS

L'équipe Dev envoie un email à l'équipe Ops pour informer que la version XX est prête sur docker.hub — ou bien via vis-à-vis. Aucune automatisation dans ce processus de notification.

⚠️ Goulots d'étranglement identifiés

🔴 Goulot #1 — Scan Trivy trop tardif

Le scan de sécurité Trivy est effectué après la publication de l'image sur Docker Hub et le pull par l'équipe Ops. En cas de CVE détectée, un aller-retour complet est nécessaire, causant des délais importants.

🔴 Goulot #2 — Deux bases de données

Le développement utilise HyperSQL (embarquée) tandis que la production tourne sur PostgreSQL. Cette divergence peut générer des incompatibilités SQL et des comportements inattendus en production.

🔴 Processus #1 — Déploiement 100% manuel

Aucune automatisation entre le push Git et le déploiement. L'équipe Ops reçoit une notification manuelle (email) et effectue le pull, le scan et le déploiement manuellement. Risque d'erreurs humaines élevé.

🔴 Processus #2 — Qualité de code non définie

La section "Qualité de code" côté frontend est marquée XXX — aucun outil ni standard de qualité n'est en place (pas de linter, pas d'analyse statique, pas de seuil de couverture).
