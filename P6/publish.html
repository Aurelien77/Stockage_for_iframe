💡 Points intéressants à noter
🔑
Variables auto vs. manuelles
Pour publish, CI_REGISTRY_USER et CI_REGISTRY_PASSWORD sont injectés automatiquement par GitLab CI. Seul GL_TOKEN pour la release doit être configuré manuellement.
🔀
Release : main seulement, jamais sur feature/*
Le stage release ne s'exécute que sur la branche main. Les branches staging et developpement bénéficient du publish mais pas du tagging SemVer.
📌
Tag :SHA vs tag :branch — lequel utiliser ?
Le tag SHA :a3f7c2d épingle un build précis et immuable — idéal en prod. Le tag branche :main pointe toujours le dernier build — pratique en staging.
⚙️
imagePullSecrets requis en staging Kubernetes
Pour que Kubernetes puisse tirer les images depuis le registry GitLab privé, un secret imagePullSecrets doit être créé dans le namespace via kubectl create secret docker-registry.
