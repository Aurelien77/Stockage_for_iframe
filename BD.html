Conception de la base de données avec Entity Framework Core

La base de données a été conçue à l’aide de Entity Framework Core, l’ORM officiel de .NET, afin de garantir une structure claire, sécurisée et maintenable.
1️⃣ Pourquoi utiliser Entity Framework Core (ORM)

Entity Framework Core permet de faire le lien entre le code C# et la base de données relationnelle.

  • Abstraction de la base de données
  • Moins de SQL manuel
  • Réduction des erreurs humaines
  • Code plus lisible et maintenable
ORM moderne
2️⃣ Rôle du DbContext

La classe DbContext représente le point central d’accès à la base de données.

  • Gestion de la connexion à la base
  • Suivi des entités (tracking)
  • Transaction et cohérence des données

Elle permet une séparation claire entre la logique métier et la persistance.

Architecture propre
3️⃣ DbSet : représentation des tables

Le DbSet<Users> représente la table Users dans la base de données.

  • Correspondance directe entre classe et table
  • Requêtes LINQ typées
  • Protection contre les injections SQL
Sécurité & typage fort
4️⃣ Configuration explicite avec OnModelCreating

La méthode OnModelCreating permet de définir précisément les règles de la base.

  • Clé primaire explicite
  • Index uniques sur Email et Login
  • Contraintes de longueur
  • Champs obligatoires

Ces règles garantissent l’intégrité des données dès le niveau base.

Intégrité des données
5️⃣ Sécurité et cohérence des données

Les choix effectués renforcent la sécurité :

  • Unicité des emails et logins
  • Limitation des tailles de champs
  • Valeur par défaut pour la date de création

Ces protections évitent incohérences, doublons et abus.

Base fiable
6️⃣ Avantages en contexte entreprise

Cette approche est parfaitement adaptée à un environnement professionnel :

  • Évolutivité facile (nouvelles tables, relations)
  • Compatibilité avec les migrations
  • Maintenance simplifiée
  • Standard reconnu dans l’écosystème .NET
