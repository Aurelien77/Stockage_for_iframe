Sécurité & Authentification Backend

La sécurité de l’application repose sur plusieurs mécanismes complémentaires : hashage des mots de passe, JWT et cookies sécurisés.
1️⃣ Hashage des mots de passe

Les mots de passe ne sont jamais stockés en clair en base de données.

  • Le mot de passe est transformé via une fonction de hash
  • Le résultat est irréversible
  • Même en cas de fuite de la base, les mots de passe restent protégés

Lors du login, le mot de passe saisi est hashé puis comparé au hash stocké.

Bonne pratique essentielle
2️⃣ Hashage vs Chiffrement
  • Hashage : transformation irréversible (mots de passe)
  • Chiffrement : transformation réversible avec une clé

Les mots de passe sont hashés, pas chiffrés, afin qu’ils ne puissent jamais être retrouvés.

Sécurité par conception
3️⃣ JSON Web Token (JWT)

Le JWT est utilisé pour représenter l’identité de l’utilisateur après authentification.

  • Token signé côté serveur
  • Contient l’identifiant utilisateur
  • Possède une date d’expiration

Il évite de stocker une session côté serveur.

Authentification stateless
4️⃣ Stockage du JWT dans un cookie sécurisé

Le token est stocké dans un cookie HTTP sécurisé.

  • HttpOnly : inaccessible en JavaScript (protection XSS)
  • Secure : transmis uniquement via HTTPS
  • SameSite=Strict : protection contre CSRF
  • Expiration automatique
Protection renforcée
5️⃣ Intérêt côté backend de recevoir le cookie

À chaque requête HTTP, le navigateur envoie automatiquement le cookie au backend.

  • Pas besoin d’envoyer manuellement le token
  • Le backend valide le JWT via un middleware
  • Accès contrôlé aux routes protégées

Cela simplifie la logique côté client et renforce la sécurité globale.

Backend centré sécurité
6️⃣ Avantages en contexte professionnel
  • Respect des standards de sécurité web
  • Scalable (stateless)
  • Facilement maintenable
  • Compatible frontend web & mobile

Cette architecture est largement utilisée dans les applications modernes.
