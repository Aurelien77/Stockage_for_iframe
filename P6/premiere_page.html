Communication manuelle entre DEV et OPS
L'équipe Dev envoie un email à l'équipe Ops pour informer que la version XX est prête sur docker.hub — ou bien via vis-à-vis. Aucune automatisation dans ce processus de notification.
⚠️ Goulots d'étranglement identifiés
🔴 Goulot #1 — Scan Trivy trop tardif
Le scan est effectué après pull par l'équipe Ops. En cas de CVE, un aller-retour complet est nécessaire, causant des délais importants.
🔴 Goulot #2 — Deux bases de données
Dev utilise HyperSQL (embarquée), la production tourne sur PostgreSQL. Cette divergence peut générer des incompatibilités SQL.
🔴 Processus — Déploiement 100% manuel
Aucune automatisation entre le push Git et le déploiement. L'équipe Ops effectue tout manuellement. Risque d'erreurs humaines élevé.
